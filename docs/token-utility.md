
# Token Utility y Token de Gobernanza  
alemty.eth — Tokenomics Canónicos v1.0

> Este documento define el diseño económico, la emisión, el uso y las restricciones del  
> **Token Utility / Token de Gobernanza de alemty.eth**.  
> Es vinculante y complementario al documento **Sistema Dharma · Aura · Karma**.

---

## 1. Naturaleza del Token

El Token de alemty.eth es un **token de utilidad y gobernanza**, diseñado como una herramienta de coordinación social y organizacional dentro de una DAO Web3 nativa.

El token:
- ✅ Puede ser listado en exchanges
- ✅ Puede comprarse y venderse libremente
- ✅ Puede intercambiarse entre usuarios
- ✅ Puede utilizarse en pools, DEX y mecanismos DeFi
- ❌ No representa equity
- ❌ No promete retorno financiero
- ❌ No garantiza apreciación de valor
- ❌ No otorga experiencia, rango ni reputación

> El token **no mide mérito**.  
> El token **no define estatus social**.

---

## 2. Relación con Aura y Dharma

El ecosistema de alemty.eth separa de forma estricta **experiencia, valor social y valor económico**.

| Elemento | Función |
|--------|--------|
| **Aura (XP)** | Experiencia pura y progresión de rango |
| **Dharma** | Valor social interno transferible |
| **Token** | Valor económico interoperable |

### Regla de equivalencia
- **1 Dharma = 1 Token de Gobernanza**

Esta equivalencia permite que el valor social generado dentro de la plataforma pueda interactuar con mercados externos **sin romper la meritocracia interna**.

---

## 3. Swaps Permitidos y Prohibidos

### ✅ Swaps permitidos
- Dharma → Token  
- Token → Dharma  

Estos swaps se realizarán:
- Mediante pools internos o DEX
- Bajo reglas públicas
- Con parámetros gobernables por la DAO

### ❌ Swaps prohibidos
- Token → Aura
- Dharma → Aura

> **No existe Pay‑to‑Level. Nunca.**  
> Comprar tokens **no permite** subir de rango.

---

## 4. Obtención y Uso del Dharma vía Token

Los usuarios pueden:
- Comprar Token en exchanges
- Intercambiar Token por Dharma dentro del ecosistema

El Dharma obtenido por compra:
- ✅ Puede donarse a otros usuarios
- ✅ Puede recompensar posts
- ✅ Puede consumirse en contenido premium
- ✅ Puede utilizarse en subastas y activaciones
- ❌ **No genera Aura**
- ❌ **No incrementa experiencia**
- ❌ **No sube rangos**

> El Dharma comprado **es valor**, no mérito.

## 5. Emisión del Token

### Límite máximo
- **Supply máximo teórico: 1,000,000,000 tokens (1B)**

Este límite es **contable**, no implica emisión inmediata.

---

### Emisión sostenible
- No existe emisión masiva inicial
- No existe ICO
- No existe preventa
- La emisión es:
  - Progresiva
  - Basada en interacción real
  - Vinculada a la actividad del ecosistema
  - Decreciente en el tiempo

El token se emite como consecuencia de:
- Generación de Dharma
- Actividad real de usuarios
- Crecimiento orgánico de la comunidad

> El token **no se imprime para financiar promesas**,  
> se libera para **coordinar una realidad viva**.

---

## 6. Circulación y Quema

- El token está diseñado para **circular**, no para acumularse
- Muchas acciones implican:
  - Consumo parcial
  - Quema controlada
- Esto mantiene:
  - Presión deflacionaria moderada
  - Equilibrio entre emisión y uso
